# Mapsoo Worldsmith v0.1.0-alpha.2

## 中文

这是 Mapsoo Worldsmith 的第二个公开 alpha 版本，重点把“生成过”升级为“可验证地由受信任 runner 生成”。默认网页导出已经切换到 alpha.2，同时保留 alpha.1 的历史 fixture 与验证策略，不修改既有公开哈希。

### 已验证变化

- 默认工作台导出 `mapsoo-sunny-meadow-v0.1.0-alpha.2.zip`；
- 素材 ZIP 固定为 12 个文件：1 个 manifest 与 11 个 payload records；
- 新增 `generation-receipt.json` 0.2.0 与随包 receipt schema；
- receipt 绑定真实 World Spec 字节哈希、seed、provider 执行模式、workflow、变换、AI 披露和 CC0 notice；
- Node 发布验证器按版本选择策略，未知或跨版本策略在读取包内容前失败；
- 同一个可信 generation run 连续两次浏览器导出的 ZIP 字节一致；
- 固定的纯 JavaScript PNG 编码器消除浏览器原生 PNG 压缩差异；浏览器 ZIP 与规范发布 ZIP 的路径排序、1980 固定时间、UNIX 权限和 DEFLATE level 9 编码一致；
- CI 会在真实浏览器中运行当前 exporter，并把原始 ZIP 字节与注册发布包逐字节比较；
- 固定 Sunny Meadow 公开包 SHA-256：`8c7720a8578cdc276ff69677ed0d64d8a1524d32fd00da0ffb8035b5a52bfcb6`；
- tag [release workflow](https://github.com/babyrush0101-source/mapsoo-kids/actions/runs/29674040991) 把同一个固定哈希包交给 [Godot 4.3](https://github.com/babyrush0101-source/mapsoo-kids/actions/runs/29674040991/job/88158028238) 与 [Godot 4.7](https://github.com/babyrush0101-source/mapsoo-kids/actions/runs/29674040991/job/88158028241)，并验证 `pack_id=sunny-meadow cells=384 props=29`；
- alpha.2 发布附件新增 receipt schema，不复用 alpha.1 的证据视频或成品宣传图。

### 安全与许可

素材 ZIP 仅包含 PNG、JSON 与 Markdown，不携带 GDScript 或 addon。程序化 PNG、预览和生成地图采用 CC0-1.0；schema 与说明文档的完整 MIT notice 随包写入 `license-assets.md`。Web 分发包另附 pako 1.0.11 的 MIT 与 Zlib 条款；源代码和单独安装的 importer 同样采用 MIT。包内哈希证明文件自洽，不等同于发布者身份验证。

### 10 分钟 Godot 首次导入

下载 [Sunny Meadow 素材包](https://github.com/babyrush0101-source/mapsoo-kids/releases/download/v0.1.0-alpha.2/mapsoo-sunny-meadow-v0.1.0-alpha.2.zip) 与独立的 [Godot importer](https://github.com/babyrush0101-source/mapsoo-kids/releases/download/v0.1.0-alpha.2/mapsoo-godot-importer-v0.1.0-alpha.2.zip)，然后按 [`10_FIRST_GODOT_IMPORT.md`](../10_FIRST_GODOT_IMPORT.md) 完成固定哈希校验、插件启用和场景导入。成功或失败均可通过[结构化反馈表单](https://github.com/babyrush0101-source/mapsoo-kids/issues/new?template=first-import-feedback.yml)提交。

## English

This second published alpha upgrades “generated” into “verifiably produced by the trusted runner.” The default web export now targets alpha.2 while the published alpha.1 fixture, verifier, and immutable hashes remain unchanged.

### Verified changes

- The default workbench exports `mapsoo-sunny-meadow-v0.1.0-alpha.2.zip`.
- The asset ZIP contains exactly 12 files: one manifest and 11 payload records.
- `generation-receipt.json` now uses contract 0.2.0 and ships with its receipt schema.
- The receipt binds actual World Spec bytes, seed, provider execution mode, workflow, transformations, AI disclosure, and the CC0 notice.
- Node release verification dispatches by trusted version policy and rejects unknown or cross-version policies before reading pack bytes.
- Two exports from the same trusted generation run are byte-identical.
- A pinned pure-JavaScript PNG encoder removes native browser compression drift; browser and release ZIPs share path order, fixed 1980 timestamps, UNIX permissions, and DEFLATE level 9 encoding.
- CI executes the current exporter in a real browser and compares its raw ZIP bytes with the registered release pack.
- Pinned Sunny Meadow public SHA-256: `8c7720a8578cdc276ff69677ed0d64d8a1524d32fd00da0ffb8035b5a52bfcb6`.
- The tag [release workflow](https://github.com/babyrush0101-source/mapsoo-kids/actions/runs/29674040991) passes that exact fixed-hash pack to [Godot 4.3](https://github.com/babyrush0101-source/mapsoo-kids/actions/runs/29674040991/job/88158028238) and [Godot 4.7](https://github.com/babyrush0101-source/mapsoo-kids/actions/runs/29674040991/job/88158028241), requiring `pack_id=sunny-meadow cells=384 props=29`.
- Alpha.2 adds the receipt schema attachment and does not reuse alpha.1's evidence video or finished promotional images.

### Security and licensing

The asset ZIP contains only PNG, JSON, and Markdown—no GDScript or addon. Procedural PNGs, the preview, and generated map assets are CC0-1.0. Its `license-assets.md` carries the complete MIT notice for bundled schemas and documentation. The web distribution separately carries pako 1.0.11's MIT and Zlib terms; source and the separately installed importer are also MIT-licensed. In-pack hashes prove consistency, not publisher identity.
