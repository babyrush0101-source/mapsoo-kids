# Mapsoo Worldsmith v0.1.0-alpha.8

Alpha.8 adds a reproducible, no-UI bridge from a public-safe STOYO scene request to an executable-free Godot world pack and a machine-readable request-to-pack receipt.

## New workflow

```bash
pnpm stoyo:export -- \
  --input examples/integrations/stoyo/river-valley-asset-request.json \
  --out-dir ./stoyo-output \
  --completed-at 2026-07-19T12:00:00.000Z
```

The command requires Node.js 20+, pnpm 11+, and Chrome/Chromium. It validates the strict `StoyoAssetRequest 1.0`, binds its canonical SHA-256, migrates the projection to World Spec 0.3 without inventing places or structures, runs the local procedural provider, and emits:

- `mapsoo-river-valley-observation-v0.1.0-alpha.7.zip`;
- `mapsoo-river-valley-observation-stoyo-export-receipt.json`.

The fixture is byte-reproducible at ZIP SHA-256 `0ef0dd3a5b76e832a31847a120f65a4a6b9a60f8c4fa7c22397762ad2b542ad8`; its canonical request SHA-256 is `ea279ebbfd3c12693469472fbca6bbc1286e07515632bd5e34b7bf698602a144`.

The release bundle includes the strict STOYO request schema, export-receipt schema, and synthetic river-valley request as standalone audited attachments. The generated bridge ZIP is intentionally reproduced by the CLI/CI rather than registered as a fourth official world-gallery asset.

## Version boundary

Alpha.8 is a toolchain and integration release. It deliberately emits the published Alpha.7 Pack Schema 0.5 contract instead of pretending that the CLI created a new asset format. The three Alpha.7 world packs remain byte-identical compatibility fixtures and retain their original filenames, archive roots, versions, and public hashes. The Alpha.8 web/tool and Godot importer versions advance independently.

## Safety and verification

- Strict UTF-8, 128 KiB pre-read size gate, duplicate-key and unknown-field rejection.
- Full cross-check of STOYO world/scene fields, ordered tags, content rating, request hash, World Spec, manifest, generation receipt, and overall ZIP hash.
- Strict Draft 2020-12 external receipt schema with positive and negative validation.
- `created → unchanged`; partial, changed, symlinked, or concurrent output fails closed without overwriting existing data.
- The CLI reopens the ZIP and verifies CRC, one root, 18 files, 17 payload records, safe paths, byte counts, and SHA-256 values before publishing output.
- Linux and Windows CI import the exact generated bridge pack on Godot 4.3 and 4.7 and require `created → unchanged → conflict preserved`.

The synthetic request intentionally produces zero semantic places and zero structures because `requiredSceneTags` are not inferred into authored world semantics. This release does not claim a STOYO production consumer, external adoption, Godot Asset Library publication, or independent user feedback. itch.io publication remains postponed.
