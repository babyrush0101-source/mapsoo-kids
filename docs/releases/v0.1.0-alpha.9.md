# Mapsoo Worldsmith v0.1.0-alpha.9 candidate

Status: implemented on `main`, not yet published. No tag, immutable release attachment set, release workflow URL, or release SHA-256 is claimed by this document.

Alpha.9 is the first complete reference-to-world vertical slice. One environment reference image, one character reference image, a short description, and a seed enter a local one-click workflow. The offline procedural provider emits a validated, executable-free `topdown-farm` Pack Schema 0.6.0 ZIP for a separately installed Godot 4.3+ importer.

## Implemented candidate scope

- Generation Request V2 `1.0.0` with exactly one `environment-style` and one `character` PNG/JPEG reference;
- real media-signature, byte-size, pixel-dimension, SHA-256, runtime binding, and explicit rights validation;
- the offline procedural top-down farm provider and `topdown-farm-complete-v1` completeness policy;
- terrain, water, paths/fences, crops, structures, props, layered scene data, collision, navigation, spawn data, and preview;
- a transparent character atlas with north/east/south/west `idle` and `walk` clips;
- deterministic Pack Schema `0.6.0` ZIP generation and a privacy-minimized public receipt;
- Godot 4.3+ derivation of the farm scene and player preview;
- candidate CI coverage on Linux and Windows with Godot 4.3 and 4.7.

This candidate supports only `topdown-farm`. `side-platformer`, `isometric-action`, and `layered-depth-2d` remain planned and have no compatibility promise.

## Privacy and content boundary

Reference bytes remain local and are not embedded in the pack. Local paths, source filenames, raw reference digests, the free-text description, and attribution text are excluded from the public receipt. The World ID and seed are public-safe inputs: the ID appears in the ZIP name, manifest, and README, while the receipt records the profile, seed, reference role, owned-rights declaration, license, explicit CC0 permission, and an irreversible full-request fingerprint.

Current validation checks file structure, declared media, size and dimension budgets, byte integrity, runtime binding, and rights. It does **not** perform face recognition, identity inference, OCR, trademark detection, geolocation, or content-level image sanitization. Users must locally crop or clean sensitive, identifying, textual, or branded content before selecting a file.

## License boundary

The Alpha.9 candidate accepts only user-owned reference images for which the user explicitly confirms the right to create adaptations, redistribute generated output, and dedicate that generated output under CC0-1.0. Licensed references, including attribution or share-alike inputs, are rejected instead of being silently relicensed.

Generated PNG and runtime JSON assets are dedicated under CC0-1.0 after that explicit permission. Repository source code, schemas, and documentation remain under MIT. Reference images are not included and retain their original rights. The current provider is procedural and records `contains_generative_ai: false`; future model providers must use their own provenance, disclosure, and license policy.

## Godot boundary

Pack Schema 0.6 declares Godot `4.3` as its minimum. The candidate importer derives editable terrain and scene layers, crop/prop/structure sprites, collision/navigation resources, spawn data, and an `AnimatedSprite2D` character preview. Candidate CI covers Godot 4.3 and 4.7 on Linux and Windows and requires deterministic `created → unchanged` behavior.

This is an asset and import contract, not a player controller, pathfinding implementation, farming economy, inventory, quest, combat, save system, or complete game.
